# AI 商机雷达项目蓝图

## 项目定位

AI 商机雷达不是“预测稳赚项目”的工具，而是一个帮助个人创业者、小团队、内容创作者和服务商降低选题错误率的决策系统。

它要回答三个问题：

1. 现在哪些 AI 方向正在升温？
2. 哪些方向技术热、讨论热，但产品化还不拥挤？
3. 哪些方向适合个人或小团队用较低成本验证变现？

一句话定位：

> 用多平台实时数据，筛选适合个人和小团队切入的 AI 变现机会。

## 目标用户

第一批目标用户不要太泛，优先服务这几类人：

| 用户 | 痛点 | 可卖内容 |
|---|---|---|
| 独立开发者 | 不知道做什么产品，怕做完没人买 | 机会排行、竞品拆解、MVP 路线 |
| AI 工具出海团队 | 需要跟踪 GitHub / Product Hunt 趋势 | 数据监控、周报、竞品库 |
| 内容创作者 | 需要持续找 AI 热点和选题 | 选题雷达、热点解释、案例库 |
| 咨询/服务商 | 需要给客户推荐 AI 改造方向 | 行业 Agent 模板、客户方案 |
| 电商/跨境卖家 | 需要发现 AI 相关产品机会 | 商品趋势、工具需求、供应链风险 |

## 当前系统能力

已经完成：

- 多平台准实时采集：GitHub、Product Hunt、arXiv、Hugging Face、Hacker News
- 标准化数据输出：搜索结果 CSV、产品/项目 CSV、原始快照
- 异常报告：搜索量不足、相关度不足、数据陈旧
- 机会评分：把多平台数据转成机会排行
- 可审计证据：保存 raw JSON/XML 原始响应

## 核心数据口径

| 平台 | 代表信号 | 注意事项 |
|---|---|---|
| GitHub | 开发者供给、开源热度 | 不代表用户愿意付费 |
| Product Hunt | 产品化热度、商业尝试 | 不代表真实收入 |
| arXiv | 研究前沿、技术萌芽 | 论文多不等于能落地 |
| Hugging Face Models | 模型生态、开源模型供给 | 当前轻量接口不是完整搜索总量 |
| Hugging Face Spaces | Demo / 原型生态 | 适合看产品原型，不等于付费市场 |
| Hacker News | 工程师讨论和早期需求 | 讨论热不等于大众市场 |

## 商机判断逻辑

最值得关注的不是“所有平台都热”，而是这种组合：

```text
技术供给强 + 社区讨论强 + 产品化相对不拥挤 + 个人可执行
```

典型解释：

- GitHub 很热：说明开发者正在构建。
- Hacker News 很热：说明工程师和早期用户在讨论。
- Product Hunt 不太拥挤：说明成品化空间可能还在。
- 不是硬件/供应链：说明个人或小团队更容易验证。

## 产品形态路线

### 第 1 阶段：数据和报告服务

目标：证明有人愿意为“AI 机会判断”付费。

交付物：

- 每周 AI 商机报告
- Top 10 方向排行
- 每个方向的竞品、风险、下一步验证动作
- 1-3 个适合个人执行的具体 MVP 建议

可收费：

- 单份报告：99-199 元
- 月度订阅：199-399 元
- 定制方向分析：499-1999 元

### 第 2 阶段：半自动工具

目标：把重复的数据收集和评分自动化。

功能：

- 自定义关键词
- 自动拉取多平台数据
- 机会评分表
- 异常变化提醒
- 竞品列表导出

### 第 3 阶段：决策系统

目标：从“看数据”升级到“给路线”。

功能：

- 输入个人资源：开发能力、行业资源、预算、渠道
- 输出适合方向
- 输出 7 天验证计划
- 输出落地页文案和冷启动客户列表

## 近期开发任务

优先级从高到低：

1. 增加趋势对比：保存每天结果，计算 7 天、30 天变化。
2. 增加机会回测：记录每次推荐，30 天后复盘是否命中。
3. 增加数据源：Google Trends、Reddit、YouTube、X、G2/Capterra。
4. 增加竞品详情页：价格、定位、发布时间、用户反馈。
5. 增加 Web 界面：展示排行、趋势图、证据链和报告导出。

## 风险声明

这个系统只能降低盲目决策概率，不能保证项目一定赚钱。真正的商业验证还需要：

- 用户访谈
- 付费意愿测试
- 落地页转化率
- 小范围销售
- 竞品差评分析

系统输出应该永远保留三个部分：

```text
推荐理由
反对理由
下一步验证动作
```
